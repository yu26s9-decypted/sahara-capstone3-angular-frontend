What we are building

Sahara is designed to feel familiar: search for products, view details, sign in, manage your account, add items to a cart, and check out without needing a treasure map or a group chat for emotional support.

Behind the scenes, Sahara connects an Angular frontend to a Java backend, with real API calls, real state changes, and real moments where one missing import can humble an entire afternoon.

Why Sahara exists

This project exists to demonstrate full-stack development in a realistic ecommerce setting. The goal is not to pretend we have a warehouse, a logistics fleet, or a boardroom with dramatic lighting. The goal is to build a believable shopping platform with the core pieces working together.

If this were a real company, our mission statement would probably say something about making shopping simple. Since this is also a capstone, our unofficial mission is: make shopping simple, make the code understandable, and make sure the cart count does not lie.

Useful

Features should help users do the next obvious thing without hunting for it.

Honest

Prices, totals, loading states, and errors should be clear. Mystery belongs in novels, not checkout.

Polished

The interface should feel calm, consistent, and slightly more organized than the average project folder.

How we think about customers

People should be able to understand Sahara quickly. Product pages should explain what is being sold. Account pages should not feel like paperwork. Checkout should move confidently, but not aggressively, because nobody likes being rushed by a button.

We also believe loading states matter. If the database is thinking, the interface should say so. Silence during an API call is how users start clicking things twice and developers start whispering to logs.

A quick note from the team

Sahara is still a learning project, so some parts may be sample data, classroom architecture, or features that are one good commit away from feeling fancy. But the intent is real: build a store that looks believable, behaves predictably, and proves the stack can work.
